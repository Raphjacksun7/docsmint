## Install

```bash
pip install artifacts-keyring
pip install mkdocx \
  --index-url https://centiro.pkgs.visualstudio.com/_packaging/Internal_Python/pypi/simple/ \
  --extra-index-url https://pypi.org/simple/
```

<Callout type="note">
`artifacts-keyring` handles authentication with the internal Azure Artifacts feed automatically.
</Callout>

## Prerequisites

- Python 3.11+
- Node.js 18+ and pnpm (or npm) — used internally by mkdocx to run Astro

## Initialize a project

From your project root:

```bash
mkdocx init
```

This creates:

```
docs/
├── mkdocx.config.js
└── src/
    └── content/
        └── docs/
            └── getting-started.md
```

## Start the dev server

```bash
mkdocx dev
```

On first run, mkdocx installs Node.js dependencies automatically. The dev server starts at `http://localhost:4321`.

## How it works

mkdocx is a "fat CLI" — the Astro rendering engine is bundled inside the pip package. Your repo only contains content and config. The CLI:

1. Copies the bundled template to `docs/.mkdocx/`
2. Symlinks your `mkdocx.config.js` and `src/content/` into it
3. Runs Astro from `docs/.mkdocx/`

You never touch the rendering system. Updates arrive via `pip install --upgrade mkdocx`.

## Commands

| Command | Description |
|---------|-------------|
| `mkdocx init` | Scaffold `docs/` in the current project |
| `mkdocx dev` | Start dev server at `localhost:4321` |
| `mkdocx build` | Production build → `docs/.mkdocx/dist/` |
| `mkdocx preview` | Preview the production build locally |
| `mkdocx clean` | Remove `docs/.mkdocx/` to force a fresh reinstall |
